## Key Insight: Old Languages = Architecture Revelation

The constraint engine's natural shape IS what COBOL and Fortran force:
- Fixed-size records (no dynamic allocation in hot path)
- Linear pipeline of sections (no branching on type)
- In-place mutation (no copies, no return values)
- Column-major array access (cache-friendly by default)
- Iterative BFS on bounded queue (no recursion, no heap)

Every modern language feature that obscures these facts (dynamic sizing, heap allocation, iterator chains, trait objects, Result types) makes code easier to write and harder to understand.
